The job of the application toolkit is to mesh the various WinRT application events (PLM, Activation, etc.) with DOM events (DOMContentLoaded, unload, etc.) with user code. To understand the technical implementation (as opposed to the end developer model), you need to understand two concepts - event queue and dispatch. An event is queued to the application object based on some external stimulus. Events are dispatched after any previous events have been dispatched (first in, first out rules). No event is dispatched until "start" is called on the application. By queueing events until the application code is ready to receive them, we put the developer in control of when they want to start receiving these various events.
